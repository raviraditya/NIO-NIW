# NIO-NIW

MATLAB analysis framework for investigating tropical cyclone–generated near-inertial wave energetics and vertical energy redistribution in the North Indian Ocean.

This repository accompanies the manuscript:

> Singh, R., & Behera, M. R. (2025). *Tropical Cyclone Generated Near-Inertial Energy and Its Vertical Redistribution in the North Indian Ocean*.  
> JGR: Oceans (submitted).

---

## Repository Contents

### Core Analysis

- `Main_combined.m`  
  Primary diagnostic engine for per-cyclone processing of HYCOM ocean-state outputs. Computes:
  - near-inertial velocity decomposition
  - near-inertial kinetic energy (NIKE)
  - WKB-normalised energetics
  - vertical group velocity
  - wind-power input
  - resonance parameters
  - right/left TC-relative partitioning

  Set `STORM_ID` to one of:
  `KYARR`, `AMPHAN`, `FANI`, `TAUKTAE`.

---

### Post-processing

- `Post_combined.m`  
  Reads the `.mat` and `.xlsx` outputs generated by `Main_combined.m` and computes:
  - penetration-depth diagnostics
  - energy-conversion efficiency
  - Richardson-number statistics
  - right/left asymmetry metrics
  - Langmuir-turbulence indicators

---

### HYCOM–Argo Validation

- `HYCOM_Argo_validation.m`  
  Validates HYCOM upper-ocean temperature, salinity, and mixed-layer depth against quality-controlled Argo profiles.

  Features:
  - spatiotemporal colocation (50 km / 3 h)
  - QC filtering using Argo quality flags
  - support for delayed-mode adjusted variables
  - temperature/salinity validation from 5–500 m
  - mixed-layer-depth comparison
  - Murphy Skill Score computation

  Produces the statistics reported in:
  - Text S1
  - Figure S2
  - Table S1

  of the Supporting Information.

## Requirements

- MATLAB R2021a or newer

## Data Sources

### HYCOM

HYCOM GLBy0.08 reanalysis outputs were used for upper-ocean diagnostics and validation analyses.

Source:
https://www.hycom.org/dataserver/gofs-3pt1/reanalysis

---

### ERA5

ERA5 atmospheric forcing fields were obtained from the Copernicus Climate Data Store (CDS).

The original ERA5 files were downloaded in GRIB format and converted to NetCDF format prior to preprocessing and analysis. No spatial regridding was applied during conversion.

---

### Argo

Argo delayed-mode profile observations were obtained from the Argo Global Data Assembly Centre (GDAC).

Source:
https://argo.ucsd.edu/

---

### Cyclone Tracks

Cyclone best-track data were obtained from the India Meteorological Department (IMD) cyclone e-Atlas.

---

## Expected Data Layout

Input datasets should be organised under:

```text
data/
  IMD/          # cyclone best-track CSVs
  HYCOM/        # per-TC HYCOM NetCDF snapshots
  Stress/       # ERA5 surface stress NetCDF
  Wind/         # ERA5 10-m wind NetCDF
  Currents/     # ocean-current NetCDF (optional)
  INCOIS_OHC/   # ocean heat-content NetCDF (optional)
  Argo/         # Argo profile NetCDFs
```

## Workflow

### Main Analysis

1. Set `STORM_ID` inside `Main_combined.m`
2. Run `Main_combined.m`
3. Run `Post_combined.m`

---

### HYCOM–Argo Validation

To reproduce Supporting Information validation analyses:

```matlab
HYCOM_Argo_validation
```

This validation workflow is independent of the main NIW analysis pipeline.

---

## Notes

- Argo pressure (dbar) is treated approximately as depth (m) for interpolation and mixed-layer-depth estimation.
- Mixed-layer depth is computed using the 0.2°C threshold criterion following de Boyer Montégut et al. (2004).
- Longitude conventions are internally converted to HYCOM-compatible coordinates where required.
